Téléchargement

La dernière version de Danger from the Deep est la 0.3.0. La simulation est jouable mais est toujours en cours de développement actif et est donc considéré à l'état de version alpha mais vos idées, rapports de bogues et suggestions participeront au façonnement des futures versions de DftD et nous serons heureux de recevoir vos remarques, même s'il s'agit pour vous de nous dire que DftD fonctionne sur votre ordinateur.

Pré-requis

Pour jouer à Dangerdeep sur l'un des systèmes d'exploitation supportés, vous aurez besoin de :

  • Une carte graphique compatible OpenGL 1.5 (OpenGL 2.0 ou plus recommendé)
  • Un processeur assez rapide, entre 1.0ghz et 1.5ghz ou plus
  • 256 Mo de RAM (512 Mo ou plus rendra la simulation plus agréable)
  • Les périphériques habituels : clavier, écran, souris et des hauts-parleurs ou des écouteurs

Paquets "tout en un"

  • Windows (win32):- dangerdeep-setup-0.3.0.0.exe
  • GNU/Linux (x86 & x86-64):- dangerdeep-0.3.0-linux-installer.bin

Les paquets ci-dessus contiennent à la fois le programme et les données. L'ensemble est placé sous licence GPL et Creative Commons respectivement (cf. le contenu des Licences pour plus de détails). Si vous ne souhaitez pas utiliser ces installateurs, vous pouvez télécharger les binaires ou les sources ci-dessous mais vous devrez également télécharger les données.

Paquets individuels

  • Code source :- dangerdeep-0.3.0.tar.gz
  • Données :- dangerdeep-data-0.3.0.zip
  • Paquet Debian (unstable) :- dangerdeep_0.3.0.1_i386.deb (mis-à-jour)
  • Pour Gentoo, merci d'utiliser un port (~x86 et ~amd64).

Platformes/distributions ayant besoin de support

  • (vieux) port FreeBSD :- dangerdeep-0.2.0.tbz
  • (Très vieux) Mandrake(10.2) :- dangerdeep-0.1.0-1mdk.i586.rpm
  • (Très vieux) RPM générique :- dangerdeep-0.1.0-1mdk.src.rpm

Les paquets avec l'indication (vieux) sont orphelins et attendent des bonnes volontés pour les mettre à jour, leur utilisation est donc déconseillée.

CVS

Si vous avez l'âme d'un aventurier et si vous cherchez à compiler la dernière version à partir de l'environnement de développement, vous pouvez obtenir le code source et les données en exécutant les commandes suivantes :

cvs -d:pserver:anonymous@dangerdeep.cvs.sourceforge.net:/cvsroot/dangerdeep login

Lorsque le mot de passe vous est démandé, il suffit de valider (compte anonyme). Pour télécharger les fichiers :

cvs -z3 -d:pserver:anonymous@dangerdeep.cvs.sourceforge.net:/cvsroot/dangerdeep co -P dangerdeep

Vous allez alors voir un certain nombre de lignes s'afficher (une ligne par fichier récupéré depuis le serveur). Lorsque vous récupérez le prompt, vous devriez avoir un répertoire "dangerdeep". Pour les mises-à-jour ultérieures de cette arborescence, assurez-vous que vous êtes dans le répertoire dangerdeep et lancez :

cvs update -Pd

Pour la compilation, nous utilisons l'outil scons. Une version debug de dangerdeep peut être obtenue par la commande :

scons debug=1 usex86sse=-1 datadir=/path/to/cvs/dangerdeep/data

Quelques explications sur les options utilisées :

  • debug=1 Activation du mode debug (valeurs possibles : 0 à 3)
  • usex86sse=-1 Désactive toutes les instructions SSE
  • datadir=/path/to/cvs/dangerdeep/data Ceci est le répertoire où vous avez placé le répertoire /data téléchargé depuis le serveur. Si cette variable pointe vers un répertoire inexistant ou non à jour, dangerdeep ne fonctionnera pas correctement et s'arrêtera en erreur.

Avertissement

La version CVS est une version de DEVELOPPEMENT, tout peut donc arriver : les poules auront des dents, un éléphant rose peut survoler votre maison, votre chat peut même se transformer en une souris blanche. Vous devez vous attendre à toute manifestation de phénomènes pas très naturels. Enfin, cela ne devrait pas arriver, mais vous êtes prévenus !
